# Wild-fixture interop (local, never committed)

Interop validation against implementations **other than zarr-python and libzarr**. No test
data is committed to this repo (see the test-data rule in `.claude/CLAUDE.md`): the scripts
here generate stores locally, and the generated bytes are gitignored. CI regression for the
specific quirks these surface is handled by synthetic unit tests (`tests/test_*.cpp`) and
the zarr-python conformance harness, which writes its fixtures fresh on every run.

## Generators

Each script writes small stores plus a `manifest.json` (crc32c + byte count of every fully
decoded array, computed by the *writing* library) into a local directory:

- `tensorstore.py` — [TensorStore](https://github.com/google/tensorstore): v2 gzip/blosc,
  v3 zstd, v3 sharded (gzip inner chunks + crc32c index). Needs `pip install tensorstore`.
- `gdal.py` — GDAL's Zarr driver: v2 (zlib) and v3 (gzip). Needs a GDAL-enabled Python.
- `nczarr.py` — libnetcdf's NCZarr (`pip install netCDF4`): both `zarr` and `nczarr` modes,
  exercising the string-typed numbers and shuffle filter that genuine NCZarr writes.

```sh
python3 tests/wild/tensorstore.py /tmp/wild/tensorstore
python3 tools/make_wild_manifest.py /tmp/wild/tensorstore   # if the script didn't
./build/conformance_tool verify-manifest /tmp/wild/tensorstore
```

`conformance_tool verify-manifest <dir>` decodes every array with libzarr and checks its
crc32c against the manifest — a bit-for-bit cross-check against the foreign writer.

## Public-store survey and full-data checks

`tools/survey.py` (metadata, wide net) and `tools/fetch_dataset.py` (complete arrays, real
chunk data) probe live public stores over HTTP. These are manual verification runs — run
them for real when checking interop, but never commit what they download. Results of past
runs are recorded in [SURVEY.md](SURVEY.md).
