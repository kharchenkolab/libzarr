# Wild fixtures

Small Zarr stores written by implementations **other than zarr-python and libzarr**, used
as interop regression tests. Each store carries a `manifest.json` (generated by
`tools/make_wild_manifest.py`) recording the crc32c and byte count of every fully decoded
array as computed by zarr-python; the `wild_*` ctest entries verify that libzarr decodes
the same bytes.

## tensorstore/

Written by [TensorStore](https://github.com/google/tensorstore) (Google's C++
implementation), version 0.1.x, via the script in the repo history: v2 arrays with gzip and
blosc compressors, a v3 array with zstd, and a v3 sharded array with gzip inner chunks and
a crc32c-checked index.

## gdal/

Written by GDAL 3.13.1's Zarr driver (`tests/wild/generate/gdal.py`): a v2 store (zlib) and
a v3 store (gzip), both with nodata/fill and non-square blocks.

## nczarr/

Written by libnetcdf 4.9.3's NCZarr (`tests/wild/generate/nczarr.py`), in both pure-`zarr`
and `nczarr` modes. These stores carry genuine quirks: numbers as JSON strings
(`"level": "1"`, `"elementsize": "0"`) and shuffle filters — as of this writing zarr-python
itself cannot read them (numcodecs crashes on the string elementsize), so their manifests
are computed by netCDF4 (the writing library) inside the generator script.

## ome_zarr/

A pruned subset of a real public OME-Zarr image: [IDR](https://idr.openmicroscopy.org)
image **6001240** from study **idr0062A**
(`https://uk1s3.embassy.ebi.ac.uk/idr/zarr/v0.4/idr0062A/6001240.zarr`), written by
**omero-zarr 0.4.0**. IDR data is licensed
[CC BY 4.0](https://creativecommons.org/licenses/by/4.0/); this subset (group metadata and
four chunks of the smallest pyramid level — 4-D uint16, blosc/lz4, `/` dimension
separator) is redistributed with attribution under that license. Chunks not included read
as fill on both sides, so the manifest stays consistent.
